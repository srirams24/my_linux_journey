    முக்கியமான மூன்று ஃபைல் ஷேரிங் டெக்னாலஜிகள் மற்றும் அவற்றின் அடிப்படையான கமாண்டுகளைப் பார்க்கலாம்.

    1. 🚀 rsync: மாற்றங்களை மட்டும் அனுப்புதல் (Delta Transfer)

    rsync என்பது ஒரு சக்திவாய்ந்த ரிமோட் காப்பி (Remote Copy) டூல். இதை லோக்கலாவும் (உள்ளூர்லயும்) பயன்படுத்தலாம்.

    rsync-ன் சூப்பர் பவர்

    rsync-ன் மிக முக்கியமான மேஜிக் என்னவென்றால், ஒரு ஃபைலை நீங்கள் முதன்முதலில் காப்பி செய்த பிறகு, அடுத்தடுத்து காப்பி செய்யும் போது, அது சோர்ஸுக்கும் டெஸ்டினேஷனுக்கும் இடையே உள்ள மாற்றங்களை (Deltas or changes) மட்டுமே டிரான்ஸ்ஃபர் செய்யும்.

    • பயன்: மொத்த ஃபைலையும் திரும்ப அனுப்ப வேண்டியதில்லை. இதனால் நெட்வொர்க் பேண்ட்வித் (Network Bandwidth) ரொம்ப குறைவாகவே செலவாகும்.
    • எடுத்துக்காட்டு: ஒரு பெரிய ஃபைலில் ஒரு சின்ன மாற்றம் செய்தபோது, டிரான்ஸ்ஃபர் ஆன சைஸ் வெறும் $185$ பைட்களாக மட்டுமே இருந்தது (இது மொத்த ஃபைலின் சைஸ் அல்ல).

    🔥 rsync-ன் முக்கிய கமாண்ட் ஆப்ஷன்கள் (Archive & Attributes)

    டேட்டா மற்றும் பெர்மிஷன்களைப் பாதுகாக்க, இந்த ஆப்ஷன்களை கட்டாயம் பயன்படுத்த வேண்டும்:

    ஆப்ஷன் (Option) விளக்கம் (Tamil Explanation) எதற்குப் பயன்படுகிறது? ஆதாரம்
    -A Archive ஆப்ஷன். இதுதான் மெயின். இது ஃபைலின் பெர்மிஷன்ஸ் (Permissions) மற்றும் ஓனர்ஷிப் (Ownerships) போன்ற எல்லாவற்றையும் பாதுகாக்கிறது. பெர்மிஷன்களைப் பாதுகாக்க
    -X Extended Attributes (SE Linux Context) போன்ற நீட்டிக்கப்பட்ட ஃபைல் அட்ரிபியூட்களைப் பாதுகாக்க. இது இல்லாவிட்டால் சில ஃபைல் அட்ரிபியூட்கள் டிரான்ஸ்ஃபர் ஆகாது. SE Linux Context பாதுகாக்க
    --stats டிரான்ஸ்ஃபர் நடந்ததற்கான புள்ளிவிவரங்களைக் (Statistics) காட்டும். எத்தனை ஃபைல்கள் உருவாக்கப்பட்டது/அழிக்கப்பட்டது, டிரான்ஸ்ஃபர் செய்யப்பட்ட மொத்த சைஸ் என்ன போன்ற விவரங்களைப் பார்க்கலாம். டிரான்ஸ்ஃபர் விவரங்களுக்கு
    --delete இது ரொம்ப கவனமா இருக்க வேண்டிய ஆப்ஷன். சோர்ஸ் லொகேஷனில் ஒரு ஃபைலை அழித்துவிட்டால், டெஸ்டினேஷனிலும் அது அழிய வேண்டும் என்றால் இதைச் சேர்க்க வேண்டும். rsync இயல்பாகவே டேட்டா இழப்பதைத் தவிர்க்க ரொம்ப "கன்சர்வேட்டிவ்" ஆக இருக்கும், அதனால் இதைச் சொல்லாவிட்டால் டெலிட் செய்யாது. சிங்க்ரோனைஸ் செய்யப்பட்ட ஃபைல்களை டெலிட் செய்ய

    🔒 rsync பாதுகாப்பு

    rsync கமாண்டை நாம் பயன்படுத்தும்போது, பின்னணியில் அது SSH Infrastructure-ஐ பயன்படுத்துகிறது. இதனால் டேட்டா என்கிரிப்ட் செய்யப்பட்ட சேனல் (Encrypted Channel) வழியாகவே டிரான்ஸ்ஃபர் ஆகிறது.

    2. 📁 Samba (SMB/CIFS): லினக்ஸில் விண்டோஸ் ஷேரிங்

    Samba என்பது லினக்ஸ் சிஸ்டம் மூலமாக விண்டோஸ் ஃபைல் ஷேரிங் (CIFS/SMB) சேவைகளை உருவாக்க உதவும் டூல்களின் தொகுப்பு.

    • SMB = Server Message Block. CIFS = Common Internet File Service.
    • இன்று பல NAS சாதனங்கள் மற்றும் ஸ்டோரேஜ் அரேக்கள் (Storage Arrays) அவற்றின் விண்டோஸ் ஷேர்களுக்குப் பின்னணியில் லினக்ஸையும் Samba-வையும் தான் பயன்படுத்துகின்றன.

    🛠️ கான்பிகரேஷன் (Configuration)

    பகுதி செயல்முறை / கமாண்ட் விளக்கம் ஆதாரம்
    Package Samba (சர்வர்) மற்றும் Samba-Client (கிளைன்ட்). தேவையான சாம்பா பேக்கேஜை இன்ஸ்டால் செய்யவும்.
    Service systemctl enable --now smb சர்வர் மெசேஜ் பிளாக் (SMB) சர்வீஸை இயக்கவும்.
    Config File /etc/samba/smb.conf ஷேர்களை இங்கே தான் வரையறுக்க வேண்டும்.
    SE Linux semanage fcontext -a -t samba_share_t "/srv/share_one(/.*)?" SE Linux Context-ஐ செட் செய்ய வேண்டும்.

    ⚙️ ஷேர் ஆப்ஷன்கள் (Share Options in smb.conf)

    ஷேர் கான்பிகரேஷனில் சில முக்கியமான விஷயங்கள்:

    • Path: ஷேர் செய்ய வேண்டிய ஃபைல் சிஸ்டம் லொகேஷன்.
    • Browsable: நெட்வொர்க்கில் பிரவுஸ் (browse) செய்யும்போது இந்த ஃபோல்டர் தெரியுமா, தெரியாதா.
    • Write List: ஃபைல் சிஸ்டம் பெர்மிஷன்களைத் தாண்டி, யாரெல்லாம் அந்த ஷேரில் எழுத அனுமதிக்கப்படுகிறார்கள் என்பதைக் கட்டுப்படுத்தலாம்.
    • Public: இது அங்கீகாரம் (Authentication) தேவையில்லாமல் பொதுப் பயன்பாட்டிற்கு ஷேர் செய்ய உதவுமா என்பதைக் குறிக்கும். (பொது ஷேர்களில், லினக்ஸ் nobody என்ற சலுகை இல்லாத (nonprivileged) யூசராகச் செயல்படுகிறது).
    • Hidden Shares: விண்டோஸில் செய்வது போல, ஷேர் பெயரின் முடிவில் $ குறியைச் சேர்த்தால், அது மறைக்கப்பட்ட ஷேராக (Hidden Share) செயல்படும்.

    💻 கிளைன்ட் மூலம் அணுகுதல்

    Samba-Client பேக்கேஜில் smbclient என்ற இன்டராக்டிவ் டூல் உள்ளது.

    • பயன்பாடு: இது ஒரு FTP அல்லது SFTP செஷன் போலவே வேலை செய்யும். ஃபைல்களை டவுன்லோட் செய்ய get, அப்லோட் செய்ய put போன்ற கமாண்டுகளைப் பயன்படுத்தலாம்.
    • கமாண்ட் (Shares List): smbclient -L //server_name (Target சர்வரில் உள்ள ஷேர்களைக் காண).
    • கமாண்ட் (Connect): smbclient //server_name/share_name (ஷேருடன் இணைக்க).

    3. 🌐 NFS: நெட்வொர்க் ஃபைல் சிஸ்டம்

    NFS (Network File System) என்பது சன் மைக்ரோசிஸ்டம்ஸ் (Sun Microsystems)-ஆல் உருவாக்கப்பட்ட ஒரு சிஸ்டம்.

    📝 கான்பிகரேஷன்

    • Package: NFS-utils.
    • Service: nfs-server.
    • Exports File: எந்தெந்த ஃபைல் சிஸ்டம்களை ஷேர் செய்ய வேண்டும் என்பதை /etc/exports என்ற டெக்ஸ்ட் ஃபைலில் வரையறுக்க வேண்டும்.
    • Access Control: ஷேரை யார் அணுகலாம் என்பது கிளைன்ட் IP முகவரி மூலமாகத் தீர்மானிக்கப்படுகிறது. (CIDR நோட்டேஷனிலும் கொடுக்கலாம், e.g., $10.5.1.0/24$).
    • Refresh: கான்பிகரேஷன் மாற்றப்பட்டால், exportfs -a என்ற கமாண்டைப் பயன்படுத்தி NFS-ஐ ரீஸ்டார்ட் செய்யாமல் ஷேர்களை மீண்டும் Export செய்யலாம்.

    ⚠️ முக்கியப் பிரச்சனை: ரூட் ஸ்குவாஷ் (Root Squash)

    NFS-ல் மிகவும் கவனிக்க வேண்டிய விஷயம் இது.

    நிலை விளக்கம் ஆதாரம்
    பிரச்சனை நீங்கள் சர்வருக்கு ரூட்டாக (UID 0) இருந்தாலும், NFS ஷேரில் எழுத (write) முயற்சித்தால் Permission denied என்று வரும்.
    காரணம் வெவ்வேறு சிஸ்டங்களில் உள்ள root யூசர்கள் ஒரே ஆளாக இருக்க மாட்டார்கள் என்று NFS நம்புகிறது. எனவே, ரூட் யூசரின் சலுகைகளைத் தடுத்து நிறுத்துகிறது. இதற்குப் பெயர் 'Squashing'.
    தீர்வு (Lab Environment) /etc/exports ஃபைலில் no_root_squash என்ற ஆப்ஷனைச் சேர்க்க வேண்டும். (ஆனால், இது Enterprise Deployment-க்கு பரிந்துரைக்கப்படுவதில்லை; அங்கு யூசர் ஐடென்டிட்டி (LDAP/AD) மூலம் நிர்வகிக்கப்பட வேண்டும்).

    💻 கிளைன்ட் மூலம் மவுண்ட் செய்தல் (Mounting)

    கிளைன்ட் மெஷினில், NFS-utils இன்ஸ்டால் செய்யப்பட்டிருக்க வேண்டும்.

    • கமாண்ட்: mount -t nfs <host_name>:/srv/nfs /mnt.

    🛡️ NFS பாதுகாப்பு குறிப்பு

    எளிமையான NFS (Version 3) அமைப்புகளில், டேட்டா நெட்வொர்க்கில் ப்ளெய்ன் டெக்ஸ்ட்டில் (Plain Text) செல்லும் அபாயம் உள்ளது. டேட்டா ட்ரான்ஸ்மிஷனை என்கிரிப்ட் செய்ய Kerberos போன்ற டெக்னாலஜிகளைப் பயன்படுத்த வேண்டும்.

    4. 🔏 SCP & SFTP: என்கிரிப்ட் செய்யப்பட்ட டிரான்ஸ்ஃபர்

    SCP மற்றும் SFTP ஆகிய இரண்டும் என்கிரிப்ட் செய்யப்பட்ட ஃபைல் டிரான்ஸ்ஃபர் செய்யப் பயன்படுகின்றன.

    • பின்னணி: இவை இரண்டும் Open SSH Daemon-ஐப் பயன்படுத்தி ஒரு என்கிரிப்ட் செய்யப்பட்ட சேனலை உருவாக்குகின்றன.
    • SCP: இது முன்னர் ஒரு தனி புரோட்டோகால் ஆக இருந்தது, ஆனால் இப்போது பின்னணியில் (Backends) SFTP மூலமாகவே இயங்குகிறது.
    • பழைய நெட்வொர்க்ஸ்: முற்காலத்தில் Telnet, RSH, RCP போன்ற Rtools ஆகியவை என்கிரிப்ட் செய்யப்படாத நெட்வொர்க் டிரான்சிட்டைப் பயன்படுத்தின. அதனால் டேட்டா எளிதில் திருடப்பட்டது. இப்போது rsync உட்பட பெரும்பாலான டூல்கள் SSH Infrastructure-ஐப் பயன்படுத்துவதால் பாதுகாப்பு உறுதி செய்யப்படுகிறது.
